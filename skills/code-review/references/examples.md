# Code Review Examples

Concrete before/after examples of well-formed findings.

## Reuse — Custom type guard replaced by a generated Zod enum

```ts
// equipment-types.utils.ts — hand-rolled type guard
function isEquipmentTypeColumn(value: string): value is EquipmentTypeColumn {
  return EQUIPMENT_TYPE_COLUMNS.some((column) => column === value);
}

// usage
if (sortBy && isEquipmentTypeColumn(sortBy)) {
  return { [sortBy]: sortDir ?? 'asc' };
}
```

`zEquipmentTypeScalarFieldEnumEnum` (auto-generated by Prisma + Zod) already validates the same set AND returns the parsed value typed.

```ts
// after — use the generated Zod enum
if (sortBy) {
  const parsed = zEquipmentTypeScalarFieldEnumEnum.safeParse(sortBy);
  if (parsed.success) {
    return { [parsed.data]: sortDir ?? 'asc' };
  }
}
```

**Finding:**

```
#### [critical] L18-L24 — Custom type guard reimplements generated Zod enum
`isEquipmentTypeColumn` duplicates `zEquipmentTypeScalarFieldEnumEnum`. The custom guard returns a boolean; Zod's `safeParse` also returns the typed value, removing a downstream cast.
**Suggestion:** Replace with `.safeParse()`. Delete `isEquipmentTypeColumn` and the `EQUIPMENT_TYPE_COLUMNS` constant.
```

## DRY — Duplicated fetch + transform pattern

```ts
// users-page.tsx
const { data } = useQuery(['users'], async () => {
  const res = await fetch('/api/users');
  if (!res.ok) throw new Error('failed');
  return (await res.json()).map(normalizeUser);
});

// admin-page.tsx
const { data } = useQuery(['users'], async () => {
  const res = await fetch('/api/users');
  if (!res.ok) throw new Error('failed');
  return (await res.json()).map(normalizeUser);
});
```

**Finding:**

```
#### [critical] users-page.tsx L12-L17 — Duplicates admin-page.tsx L8-L13
Identical fetch + error + normalize block. Two callers exist today; a third is likely.
**Suggestion:** Extract `useUsers()` hook in `hooks/use-users.ts` returning the typed query.
```

## KISS — Premature generic wrapper with one caller

```ts
// generic-handler.ts
export function createGenericHandler<T extends { id: string }>(
  fn: (item: T) => Promise<void>
): (items: T[]) => Promise<void> {
  return async (items) => {
    for (const item of items) await fn(item);
  };
}

// usage — only one call site
const handleAll = createGenericHandler(deleteUser);
```

**Finding:**

```
#### [warning] generic-handler.ts L4-L10 — Single-use generic wrapper
`createGenericHandler` has one caller (`users.ts:L42`) and adds nothing over a `for...of` loop.
**Suggestion:** Inline at the call site. Extract again only if a 2nd or 3rd consumer appears.
```
